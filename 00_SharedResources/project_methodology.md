# Project Methodology: "Just Fucking Do It, Get Shit Done"

Short name: `JFDI-GSD`
Last updated: 2026-03-06

This hackathon runs on extreme lean execution: maximize speed, visible progress, and demo impact.

## Core Principle

Deliver flashy, working outcomes fast. Prefer action over ceremony.

## Operating Rules

1. Speed and autonomy are prioritized over ideal process.
2. Frontend wow-factor is a top priority when there is a tradeoff.
3. Ask for clarification only when:
   - confidence is below 80%, or
   - there is potential long-term security risk.
4. Security hardening can be deferred to a later audit pass unless risk is severe.
5. Document enough for handoff at all times because work will move across tools, models, and machines.

## Task Tracking Discipline (Required)

1. Before starting work, update `task_registry.md` status to `in_progress`.
2. If work is paused/abandoned, set status to `on_hold` and add a short reason.
3. When done, set status to `done` with concise completion notes.
4. If blocked by dependency/access, use `blocked` and include the blocker.

## Anti-Loop Rule

If the same approach fails repeatedly, stop and pivot:
1. detect loop quickly,
2. try a different method/tool/model,
3. record the pivot reason in task notes.

## Quality Bar for This Mode

1. Must run.
2. Must be understandable by next agent.
3. Must be demoable fast.
4. Polish and hardening happen in follow-up passes.
