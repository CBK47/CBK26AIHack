⚡ Fallback Strategy

AUTOMINER

Earns ETH when agents are idle. Auto-switches to most profitable coin.

What This Service Does

Auto Miner is a fallback revenue engine for the swarm. If no active compute rental job is running, this node can switch into mining mode so idle hardware still generates value.

Activation Logic

  • Rental workload active: miner stays paused.
  • Rental workload idle: miner can auto-start.
  • Profitability check can rotate strategy/coin profile.

Economics Snapshot

  • Baseline profile shown: RandomX / Monero style workload.
  • Illustrative output target: around 30 kH/s class throughput.
  • Swarm goal: keep idle cycles from going to waste.

Operator Notes

  • Treat this as opportunistic fallback, not primary revenue.
  • Watch reject rate and hashrate drift for hardware issues.
  • Keep power/thermals within safe limits on long runs.

Opportunity Context

In the FREYWILL model, idle infrastructure is treated as recoverable value. Auto-Miner exists to reduce wasted runtime between paid inference and compute jobs.

Stack + Integration

  • Integrated into Project 4 service mesh and Swarm Command telemetry.
  • Payment/treasury accounting routes through the x402 shared rails.
  • Coordination with Compute Rental avoids overlap with paid workloads.

Status + Ports

  • Project 4 port: 4003 (auto-miner control plane).
  • Sibling dependencies: 4000 dashboard, 4002 compute, 4006 payment rail.
  • Service state should remain idle-first unless explicit fallback trigger fires.

Roadmap Alignment

  • Keep miner as fallback revenue, not brand-defining feature.
  • Improve profitability switching logic and thermal safety guards.
  • Preserve clear handover notes for multi-agent operation continuity.

Project Brief (From Swarm Doc)

Auto-Miner is positioned as the fallback utilization service in FREYWILL: if core AI/compute demand drops, this lane converts idle cycles into treasury-supporting revenue while keeping primary inference workloads prioritized.

  • Economic role: defensive uptime monetization, not primary go-to-market product.
  • Integration role: coordinated with compute rental and x402/tax accounting paths.
  • Operational role: safe-mode fallback with thermal and reject-rate monitoring.
  • Project 4 port map anchor: 4003 miner with 4000/4002/4006 support links.
