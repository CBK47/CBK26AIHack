The Big Three

HTML, CSS, and JavaScript

Think of a website like a person.

HTML

The Skeleton 🦴

HTML is the structure. It defines what exists on the page: headings, paragraphs, buttons, images, links.

Without HTML there is no page. It's the starting point for everything.

File: index.html

CSS

The Skin 💅

CSS controls colours, fonts, spacing, layout, animations. Without it, every website looks like a Word document from 1996.

CSS can live in a <style> tag or a separate .css file.

File: styles.css

JavaScript

The Brain 🧠

JS makes things happen. Click a button → something changes. Type in a box → get a result. It's the only language that runs in browsers.

JS can live in a <script> tag or a separate .js file.

File: script.js

💡 You technically only need HTML. A plain HTML file with no CSS or JS will display in any browser. CSS and JS are enhancements — layer them on top.

CSS

How CSS Works

CSS targets elements by selector and tells them what to look like.

/* target → what to change */ p { /* all <p> tags */ color: red; font-size: 18px; } .card { /* elements with class="card" */ background: #111; border-radius: 8px; /* rounded corners */ padding: 1.5rem; /* space INSIDE the element */ margin: 1rem; /* space OUTSIDE the element */ } #hero { /* element with id="hero" — only one per page */ min-height: 100vh; /* vh = % of viewport height */ } h1:hover { /* only when mouse is over it */ color: lime; }

The eternal struggle

How to Center a Div

This is the most Googled question in web development. Here are the three ways that actually work.

Method 1: Flexbox (recommended for almost everything)

I am centered
.parent { display: flex; justify-content: center; /* horizontal */ align-items: center; /* vertical */ height: 100%; /* parent needs a height for vertical to work */ }

Method 2: margin: auto (centers a block element horizontally)

margin: 0 auto
.child { width: 200px; /* must have a width */ margin: 0 auto; /* 0 top/bottom, auto left/right = centered */ } /* Only centers horizontally. Good for page containers. */

Method 3: CSS Grid (center anything, anywhere)

place-items: center
.parent { display: grid; place-items: center; /* shorthand for align-items + justify-items */ } /* One line. The cleanest method. */
🚫 What doesn't work: margin: auto on inline elements (span, a). Add display: block first. Also text-align: center only centers text, not the box itself.
Responsive Design

Making It Look Right on Every Screen

Responsive design means the layout responds to the screen size. Try resizing this window.

The viewport meta tag — the most important line in HTML

<meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Without this: mobile browsers render the page at ~980px then shrink it down to fit the screen. Everything is tiny. With this: the page renders at the actual device width. This one line enables all responsive CSS to work. -->

Media queries — different CSS for different screen sizes

/* Desktop: default styles (written first) */ .card-grid { grid-template-columns: repeat(3, 1fr); /* 3 columns */ } /* Tablet: screen width 768px or less */ @media (max-width: 768px) { .card-grid { grid-template-columns: repeat(2, 1fr); } /* 2 cols */ } /* Mobile: screen width 480px or less */ @media (max-width: 480px) { .card-grid { grid-template-columns: 1fr; } /* 1 col */ h1 { font-size: 2rem; } /* smaller */ }

Responsive units — stop using px for everything

px

Fixed pixels. 16px is always 16px. Use for borders, small spacing.

rem

Relative to root font size. 1rem = 16px. Scales with user's font preferences. Use for fonts.

%

Percentage of the parent. width: 50% = half the parent's width. Use for fluid layouts.

vw / vh

Viewport width/height. 100vh = full screen height. Use for full-page sections.

💡 The easiest responsive layout trick:
grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
This one line makes a grid that's automatically 3 columns on desktop, 2 on tablet, 1 on mobile. No media queries needed.
Cheat sheet

Useful Tips & Tricks

🔍 DevTools

Press F12 or Cmd+Option+I to open DevTools. Inspect any element, live-edit CSS, and debug JS.

Right-click anything → Inspect to jump straight to its HTML.

📦 The Box Model

Every element is a box. From outside in:

margin → space outside
border → the edge
padding → space inside
content → the actual thing

⚡ Flexbox in 3 lines

.parent { display: flex; gap: 1rem; flex-wrap: wrap; }

Puts children in a row with spacing. flex-wrap: wrap lets them wrap to next line.

🎨 CSS Variables

:root { --green: #00ff88; } h1 { color: var(--green); }

Define colours once at the top. Change one line to retheme everything.

📱 Quick Mobile Test

In DevTools (F12), click the phone/tablet icon (or press Ctrl+Shift+M) to toggle device emulation. Test your site at any screen size.

🔤 System Font Stack

font-family: system-ui, -apple-system, sans-serif;

Uses the OS's native font. Looks great, loads instantly, no Google Fonts needed.
